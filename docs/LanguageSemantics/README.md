# Language Semantics

This documentation explains **what the language is** and **how it behaves** and **why it works that way**.

**For end users learning to use the language**, see `/Documentation/`.
**For understanding compiler implementation**, see `/Architecture/`.

**Purpose:** Help language users and language designers understand what the language *is*, not how to compile it.

**Contents:**
- Language concepts and their semantics
- Type system design
- Evaluation strategies
- Memory model
- Exception handling
- Concurrency model
- Type system specifics (logic programming, functional aspects, etc.)
- Other core language features and their rationale (macros, modules, etc.)
