Ambiguous identifierÔÉÅ

Incorrect

Correct

 
import module { a = 1 }
import module { a = 2 }

print a

import module { a = 1 } as x
import module { a = 2 } as y

print x.a

Ambiguous identifier ‚Äòa‚Äô could refer to anonymous module import on line 1 or anonymous module import on line 2. Please remove one import or disambiguate using qualified names.

Automatic error propagationÔÉÅ

Incorrect

Correct

 
a = error "urk"
b = a + 2
main = print b

a = error "urk"
b = case a of
  (e : Exception) -> e
  a -> a + 2
main = print b

Exception ‚Äúurk‚Äù may be thrown by ‚Äòa‚Äô and automatically propagated out of ‚Äòb‚Äô. Consider adding explicit exception handling or marking with the function ‚Äòrethrowing‚Äô.

Bad indentationÔÉÅ

Incorrect

Correct

 
f =
  a = 1
	b = 2
  return (a+b)

print f

f =
  a = 1
  b = 2
  return (a+b)

print f

Mismatched indentation: line 2 uses 2 spaces, but line 3 uses 1 tab. Use consistent indentation.

Shadowing in a blockÔÉÅ

Incorrect

Correct

 
main =
  a = 1
  a = 2
  print a

main =
  a = 2
  print a

Second declaration of ‚Äòa‚Äô shadows earlier declaration. I suggest renaming or deleting one of the declarations.

Confusuable UnicodeÔÉÅ

Incorrect

Correct

 
¬µ = 1/1000000
Œº = "greek mu"
print [¬µ, Œº]

¬µ1 = 1/1000000
Œº2 = "greek mu"
print [¬µ1, Œº2]

Identifiers ‚Äò¬µ‚Äô (MICRO SIGN) and ‚ÄòŒº‚Äô (GREEK SMALL LETTER MU) look very similar in common fonts. Change them to be more visually distinct.

Default argumentsÔÉÅ

Incorrect

Correct

 
a {x=1,y=2} = x + y

main = print a

a {x=1,y=2} = x + y

main = print (a {x=1,y=2})

Function ‚Äòa‚Äô is called using default values of parameters ‚Äòx‚Äô, ‚Äòy‚Äô. Please call the function using explicit values ‚Äòa {x=1,y=2}‚Äô.

Deprecated default argumentsÔÉÅ

Incorrect

Correct

 
a {x = 2 !deprecated} = x * 3
main =
  print a

a {x = 2 !deprecated} = x * 3
main =
  print (a {x=2})

The default value of argument ‚Äòx‚Äô of function ‚Äòa‚Äô has been deprecated and may be removed in a future release. Please add the explicit value ‚Äò{x = 2}‚Äô.

Equal priority clausesÔÉÅ

Incorrect

Correct

 
clause c1
f 1 y = 1

clause c2
f x 2 = 2

prioConstrain c1 = c2

clause c1
f 1 y = 1

clause c2
f x 2 = 2

clause c3
f 1 2 = 3

prioConstrain c1 = c2 = c3

Clauses ‚Äòc1‚Äô and ‚Äòc2‚Äô have equal declared priority, but neither is more specific than the other. Furthermore they conflict for the expression ‚Äòf 1 2‚Äô. Set one to have higher priority or fix the conflict.

Failed assertÔÉÅ

Incorrect

Correct

 
a = 0
print "test"
assert (a != 0)

a = 1
print "test"
assert (a != 0)

Assertion ‚Äòa != 0‚Äô failed. ‚Äòa‚Äô received its value ‚Äò0‚Äô at line 1. You may change the assertion or ensure ‚Äòa‚Äô receives a different value.

Implicit argument not passedÔÉÅ

Incorrect

Correct

 
a k = 1
b k = k + a

print (b {k:2})

a k = 1
b k = k + a { k=k }

print (b {k:2})

Argument ‚Äòk‚Äô of function ‚Äòb‚Äô was not passed implicitly to ‚Äòa‚Äô, leading to it being undefined. I suggest passing it explicitly.

Inapplicable to typeÔÉÅ

Incorrect

Correct

 
a = i16 1
foo (n : i32) = n * 2
main =
  print (foo a)

a = i32 1
foo (n : i32) = n * 2
main =
  print (foo a)

Function foo does not have any rules for value ‚Äòi16 1‚Äô. The most similar rule is ‚Äòfoo (n : i32)‚Äô. Consider modifying the type of ‚Äòa‚Äô or adding another reduction rule.
